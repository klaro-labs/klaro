# Klaro launch-readiness QA — 2026-06-10

End-to-end QA against the production build (real Supabase + real Arc testnet
contracts), driven with Playwright + a real Rabby wallet, judged on desktop and
mobile. Method: ACT → screenshot → AUDIT against chain/DB truth; verify every
finding before fixing; re-test every fix.

## Verdict: NOT launch-ready

Every flow that can be driven autonomously is tested and every bug found is
fixed and re-verified. Three things still block a launch-ready call:

1. **Live cashout + live receipt-mint are screening-gated.** The operator
   daemon never auto-settles a payment screened in simulation (a safety
   feature — funds are never released without real screening). Without a
   screening-provider key (Chainalysis / TRM / Sumsub), settlement holds, so
   the vendor never gets a cashoutable balance and no live receipt mints. The
   *flows* are exercised in simulator mode; the *live* money-out path is gated.
2. **Cross-chain pay-in (CCTP V2) needs Base Sepolia faucet funds** to drive.
3. **The fixes below are committed but not deployed.** The live demo still
   carries them until pushed + re-verified on `klaro-peach.vercel.app`.

## Verified working (with proof)

- **On-chain money flow, real Rabby wallet:** vendor signs `createInvoice`
  (on-chain status verified) → buyer signs EIP-712 acceptance + USDC approve +
  `acceptAndPay` → invoice PAID on-chain, real USDC transferred (buyer balance
  43.89 → 38.89), `accepted_by` = buyer. Settlement then correctly holds on
  simulated screening (honest-mode).
- **Adversarial / states:** rejected signature (calm error, recoverable, no
  charge), insufficient funds (Pay blocked), invalid form input (blocked +
  hint), cross-tenant direct-URL access (RLS blocks, zero leak), paid-but-held
  invoice (honest "settling" state).
- **Dispute lifecycle:** vendor opens → operator sees (after the RLS fix) →
  assign → decide (DECIDED, outcome + reason recorded).
- **LP portal** (onboarded member), **admin console** (after RLS fix),
  **offline/PWA** (service worker serves the offline fallback), **embeds**
  (`/i` renders in a cross-origin iframe after the X-Frame-Options fix),
  **accessibility** (axe WCAG 2 A/AA: 0 serious on landing/product/pricing/
  trust/flows), **external links** (all resolve), **all public + logged-in
  pages** on desktop + mobile.
